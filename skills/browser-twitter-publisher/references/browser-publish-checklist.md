# Browser Publish Checklist

## Attach
- connect to chrome-devtools-mcp
- `list_pages`
- `take_snapshot`

## Confirm account
- open X home or profile
- verify handle before posting
- stop if redirected to login

## Research
- open search URL
- use `f=live`
- filter with `-filter:replies filter:links` when useful
- collect 10 candidates
- remove junk and duplicates

## Draft
- write one strong post
- prefer short paragraphs
- keep one main idea

## Build image
- write local SVG
- open local file in browser
- resize to target viewport
- `take_screenshot` to PNG

## Publish
- open compose URL with prefilled text
- verify composer state in snapshot
- upload image if needed
- only click `Post` when enabled
- verify fresh post URL after publish

## Failure handling
- login wall -> ask user to log in
- anti-spam / graduated access -> tell user exact blocker
- post too long -> rewrite, don’t force threads unless user wants them
- wrong account -> stop immediately
