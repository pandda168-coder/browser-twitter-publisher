# Publish Recovery

## Character overflow

If composer still shows overflow:
- remove extra hashtags first
- collapse lists into one sentence
- remove adjectives before removing facts
- keep the strongest number or contrast hook

## Wrong account risk

Before posting, verify one of:
- profile handle in left navigation
- account menu name in composer snapshot
- profile page handle

If uncertain, stop.

## X friction states

### Login wall
- ask the user to log in inside the controlled browser session
- do not continue with posting

### Graduated access / anti-spam
- report the exact message shown by X
- suggest light account warm-up activity
- avoid repeated forced posting attempts

### Composer looks ready but post does not send
- refresh snapshot
- check for dialogs or hidden friction
- verify media upload completed
- retry only once unless the user asks otherwise

## Definition of done

A post is only done when one of these is visible:
- `Your post was sent.`
- a fresh post card in timeline/profile
- the final post URL
